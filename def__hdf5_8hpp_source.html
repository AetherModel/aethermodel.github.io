134  // Lastly, we have to hijack H5open() and H5check_version(), which are called
135  // by some expanded macros of the other H5* functions. This means we can't
136  // create arma_H5open(), because we can't modify those macros. Instead, we'll
137  // create arma::H5open() and arma::H5check_version(), and then issue a using
138  // directive so that arma::H5open() and arma::H5check_version() are always
139  // called.
140  //
141  // There is potential danger in the use of a using directive like this, but in
142  // this case, I can't think of a better way to solve the problem, and I doubt
143  // this will cause problems in any situations that aren't truly bizarre. And
144  // if it does cause problems, the user can #define ARMA_DONT_USE_WRAPPER or
145  // #undef ARMA_USE_WRAPPER in their Armadillo configuration.
146  herr_t H5open();
147  herr_t H5check_version(unsigned majnum, unsigned minnum, unsigned relnum);
